Le comportement principal reste le même : affficher Hello World.
## 1. Comparaison entre `hello.js` et `hello.ts`  

### a) Code JavaScript (`hello.js`)  
```javascript  
var message1 = "Hello World";  
console.log(message1);  
```  
- `var` est utilisé pour déclarer la variable, ce qui signifie qu'elle n'a pas de restriction de type.  
- JavaScript est un langage **faiblement typé**, donc le type de `message1` est déterminé dynamiquement à l'exécution.  
- Il est possible de réassigner `message1` à une autre valeur de type différent, ce qui peut causer des erreurs.  

### b) Code TypeScript (`hello.ts`)  
```typescript  
const message1: string = "Hello World";  
console.log(message1);  
```  
- `const` est utilisé, ce qui empêche la réassignation de la variable.  
- Le typage fort est appliqué (`string`), ce qui signifie que `message1` ne peut contenir **que** une chaîne de caractères.  
- TypeScript **vérifie les types à la compilation**, réduisant ainsi le risque d'erreurs.  

## 2. Étape de transpilation  

La transpilation est le processus qui convertit le code TypeScript en JavaScript.  
En transpilation, le code TypeScript suivant :  

```typescript  
const message1: string = "Hello World";  
console.log(message1);  
```  
sera converti en JavaScript comme suit :  

```javascript  
var message1 = "Hello World";  
console.log(message1);  
```  
**Ce qui est supprimé lors de la transpilation :**  
- L'annotation de type `: string` disparaît, car JavaScript ne supporte pas le typage statique.  
- `const` peut être remplacé par `var` ou `let` selon la configuration du transpileur.  

TypeScript sert donc uniquement à **aider les développeurs lors du développement**, mais le code final exécuté dans le navigateur ou sur Node.js est du **pur JavaScript**.  

## 3. À quoi sert le typage fort dans TypeScript ?  

JavaScript étant un langage **faiblement typé**, il permet des manipulations de types dynamiques qui peuvent causer des erreurs difficiles à détecter.  
Le **typage fort** de TypeScript apporte plusieurs avantages :  

### a) **Vérification des erreurs avant l'exécution**  
- TypeScript détecte les erreurs de typage **avant** l'exécution, lors de la compilation. Il n'y a pas besoin d'attendre l'execution. 

### b) **Facilite la maintenance et l'évolution du code**  
- Dans un projet de grande envergure, les annotations de type permettent de comprendre plus facilement le rôle des variables et fonctions.  
- Une modification accidentelle d’un type peut être immédiatement repérée.
